FluentAI Documentation

Welcome to the FluentAI documentation. FluentAI is an AI-first programming language designed for building intelligent systems with uncompromising performance and safety.

Note: FluentAI is under active development. APIs and syntax may change as we approach v1.0.

Installation

FluentAI can be built from source on Linux, macOS, and Windows. You'll need Rust installed on your system.

# Clone the repository
git clone https://github.com/beamsjr/FluentAI
cd FluentAI/rust

# Build the project
cargo build --release

# The binary will be available at target/release/fluentai
Note: Native installers are coming soon. For now, please build from source.

Hello World

Let's start with the traditional Hello World program in FluentAI:

// hello.fc
$("Hello, World!").print();

To run this program:

fluentai run hello.fc

Basic Syntax

FluentAI uses a clean, expressive syntax inspired by functional programming languages:

Variables and Constants

// Immutable variable
let name = "Alice";

// Mutable variable
let mut counter = 0;
counter := counter + 1;

// Constants
const MAX_SIZE = 1000;

Functions

// Function declaration
private function greet(name: string) -> string {
    f"Hello, {name}!"
}

// Lambda expression
let double = (x) => x * 2;

// Higher-order function
let numbers = [1, 2, 3, 4, 5];
let doubled = numbers.map(double);

Pattern Matching

FluentAI features powerful pattern matching with method chaining syntax:

// Basic pattern matching
result.match()
    .case(Ok(value), => process(value))
    .case(Err(error), => handle_error(error))
    .get();

// Pattern matching with guards
user.match()
    .case({age, name}, => age >= 18, => f"{name} is an adult")
    .case({age, name}, => f"{name} is a minor")
    .get();

Method Chaining

Method chaining is at the heart of FluentAI's design philosophy:

// Data processing pipeline
users
    .filter(user => user.active)
    .map(user => user.email)
    .filter(email => email.contains("@company.com"))
    .for_each(email => send_newsletter(email));

// Optional chaining
user
    .profile?
    .settings?
    .notifications?
    .email_enabled;

Effect System

FluentAI uses an effect system to track and control side effects:

// Performing I/O effects
perform IO.print("Starting process...");
let data = perform IO.read_file("data.json");
perform IO.write_file("output.json", processed_data);

// Custom effect handlers
handle {
    let result = compute_with_logging();
    result
} with {
    Log.info(msg) => console.log(f"[INFO] {msg}"),
    Log.error(msg) => console.error(f"[ERROR] {msg}")
}

Concurrency

FluentAI provides multiple concurrency primitives:

Channels

// Create a channel
let ch = channel(100);

// Spawn a producer
spawn {
    for i in range(0, 10) {
        ch.send(i);
    }
};

// Consume values
for value in ch {
    $(f"Received: {value}").print();
}

Async/Await

// Async function
private async function fetch_data(url: string) -> Result {
    let response = http.get(url).await();
    response.json().await()
}

// Using async functions
async {
    let data = fetch_data("https://api.example.com").await();
    process(data);
}
